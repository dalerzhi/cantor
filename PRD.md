# Cantor - 产品需求文档 (PRD)

## 1. 产品定位与架构铁律
Cantor 是一个面向 B 端/工作室的云手机 Agent 生产级调度平台。

**核心架构铁律：**
- **脑手分离与 1:N 解耦**：绝对禁止 Agent 与单台设备的 1:1 强绑定。云手机仅作为"算力消耗品"和"末端执行器"。
- **Cantor 集合调度**：以 "Cantor" 为控制大脑实例，通过队列同时指挥 N 台云手机并行工作。
- **集中决策，分布式执行**：大模型推理和任务拆解全部在 Cantor 层（云端）完成，设备通过信令网关消费任务。遇到未知 UI 阻塞时，设备向 Cantor 发起"视觉求助"。

## 2. 核心模块需求

### 模块 A：中控后台 + 信令网关 (Backend)
- 对接云手机 IaaS 平台 API，管理设备生命周期。
- **信令网关**：基于 WebSocket，管理所有云手机长连接，实现双向实时通信。
- 维护 `Cantor 实例` 与 `设备群组` 的 N:M 动态映射关系。
- 任务队列管理，基于 Redis 实现 Cantor 任务的广播与设备的抢占式消费。

### 模块 B：Cantor 智能路由层 + Script Forge (大脑逻辑层)
- **智能中枢**：作为独立的思维中枢，统御被绑定的设备集群。
- **脚本优先匹配**：接收任务后，优先匹配已有快引擎脚本，零 token 成本下发执行。
- **慢引擎兜底**：UI 树无法定位或报错时，设备上传截图，Cantor 挂起任务，调用 VLM 分析并下发自愈指令。
- **Script Forge**：观察慢引擎的成功路径，调用 LLM 将操作历史编译为可复用的快引擎脚本。

### 模块 C：基建接入层 (Infrastructure Integrations)
- **IaaS Provider 模块**：对接外部云手机 API，负责设备实例的创建、销毁与状态同步。
- **RTC Controller 模块**：对接云手机的 RTC SDK，拉取实时画面流（供慢引擎 VLM 分析抽帧）并下发底层控制指令。
- 作为系统的原生内部模块（Native Modules），直接调用，彻底去除不必要的 MCP 协议层，极大降低音视频流流转与毫秒级触控的延迟。

### 模块 D：运营控制台 (Dashboard)
- **Cantor 管理**：创建和配置大脑实例（Persona Prompt、模型、绑定设备群组）。
- **设备舰队管理**：设备列表、状态、实时画面，支持动态划拨设备给不同 Cantor。
- **任务监控**：任务创建、执行链路追踪、日志审计。
- **脚本库**：管理 Script Forge 生成的快引擎脚本。

### 模块 E：信令客户端 (Worker)
- 部署在云手机内的轻量级进程，连接信令网关。
- 接收指令，调用本地 `uiautomator2` 执行，支持本地快引擎脚本运行。
- 上报执行结果、设备事件和异常截图。

## 3. 核心 Use Cases

### UC1: Cantor 实例与 Fleet 动态编排
**场景**：管理员需要为一个新的自动点赞业务分配资源。
1. 在 Dashboard 创建一个 Cantor 实例，配置 Persona 为"抖音自动化点赞助手"，指定主模型（如 GPT-4o）和视觉模型（如 Qwen-VL）。
2. 从全局设备池中筛选 100 台空闲设备，动态划拨给该 Cantor 实例（Fleet Mapping）。
3. Cantor 实例就绪，开始监听业务指令。

### UC2: 1:N 任务下发与分布式执行
**场景**：下发批量点赞任务。
1. 用户向 Cantor 实例发送自然语言指令："打开抖音，点赞前 5 个视频"。
2. Cantor 解析指令，将任务压入专属的 Redis 任务队列。
3. 绑定的 100 台云手机通过信令网关抢占式消费子任务。
4. Cantor 实时监控各设备执行状态，并在 Dashboard 展示整体进度。

### UC3: 快慢引擎自适应与 Script Forge 沉淀
**场景**：处理抖音版本更新导致的 UI 变化。
1. **快引擎失效**：抖音更新后，原有的"点赞"资源 ID 变更，设备的本地快引擎抛出 `ElementNotFound`。
2. **视觉求助（慢引擎）**：设备截取当前屏幕，通过信令通道上报 Cantor。
3. **VLM 修复**：Cantor 调用 VLM 服务分析截图，识别出新的点赞按钮坐标，下发点击坐标指令。设备成功执行。
4. **Script Forge 编译**：Cantor 记录这次成功的慢引擎干预路径，自动调用大模型生成新的快引擎脚本逻辑，经过沙箱验证后更新脚本库。后续设备直接使用新脚本（快引擎）执行。

## 4. 生产级非功能需求
- **无直连 ADB**：所有交互必须通过信令网关，适应严格的网络安全隔离。
- **高可用与可观测**：支持 Cantor 实例的水平扩容，提供全链路 Tracing 和 Prometheus 监控。
- **低延迟**：信令通道端到端延迟控制在 200ms 以内。
